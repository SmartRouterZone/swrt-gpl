# S20P AX6000 Port Notes

## Hardware

- SoC: MediaTek MT7986A, quad-core Cortex-A53
- RAM: 2 GiB DDR4
- Flash: eMMC 128 GB
- Switch: MediaTek MT7531AE
- Ethernet: 5x1G LAN plus 2x2.5G RTL8221B-VB-CG ports
- WiFi: MT7975N 2.4 GHz 4x4 and MT7975PN 5 GHz 4x4
- Other: USB 3.0, LEDs, reset button, M.2 socket

## Build

```sh
cd release/src-mtk-arm
make swrt-s20p-ax6000
```

Verified in a clean Ubuntu container. The build produced:

```text
release/src-mtk-arm/image/4.0.0.4_S20PAX6000_B5.3.0_50001-g7a38cd2.img
release/src-mtk-arm/image/4.0.0.4_S20PAX6000_B5.3.0_50001-g7a38cd2.trx
```

## Required Build Fixes

- Preserve `CONFIG_MMC_BLOCK` when enabling eMMC support. Only delete the exact `CONFIG_MMC` setting in `platform.mak`.
- Reuse RM-AX6000 ARM64 prebuilt objects for S20P where the source tree has no S20P-specific binary package.
- Use a S20P-specific `rc/swrtex.c` stub instead of RM-AX6000 `swrtex.o`, because the RM object references RGB/I2C LED hooks that S20P does not use.
- Replace Bash brace expansion in swrtmesh stage rules with explicit file lists so the rules work under `/bin/sh`.

## Notes

The current validation covers build completion and image generation only. Runtime LED, switch, PHY, WiFi calibration, and upgrade behavior still need device-side testing.

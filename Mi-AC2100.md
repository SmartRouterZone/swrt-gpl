# Mi AC2100 SWRT 适配记录

## 摘要

MI-AC2100 与已适配的 SWRT-RM-AC2100 同属 MT7621 + MT7603E + MT7615E 的 AC2100 平台。本次按最小风险方案适配：新增 MI-AC2100 独立构建 profile、model ID、DTS 文件和用户态识别，但板级网络、无线、分区和预编译资源暂复用 RM-AC2100 路径。

参考来源：

- 本仓库已有 `SWRT-RM-AC2100` 适配。
- OpenWrt `mt7621_xiaomi_mi-router-ac2100.dts` / `mt7621_xiaomi_redmi-router-ac2100.dts`：两者共用 `mt7621_xiaomi_router-ac2100.dtsi`，差异主要是 model/compatible 和 LED GPIO 命名。
- `lukasz1992/asuswrt-xiaomi` README：该旧 AsusWRT fork 曾同时支持 Redmi AC2100 和 Xiaomi AC2100，构建名分别对应 `rt-rm2100` / `rt-r2100`。

## 本次改动

- 新增构建目标：`SWRT-MI-AC2100`，预期命令 `make swrt-mi-ac2100`。
- 新增编译宏：`MIAC2100=y`、`MODEL_MIAC2100=y`。
- 新增 DTS：`mt7621-MI-AC2100.dts`，从 RM-AC2100 DTS 派生，只调整 `compatible` 和 `model`。
- WiFi 编译入口增加 `CONFIG_MODEL_MIAC2100`，沿用 MT7603 + MT7615 路径。
- 新增 `MODEL_MIAC2100` / `SWRT_MODEL_MIAC2100` 映射。
- `rc/shared/watchdog/www` 中 RM-AC2100 专属分支扩展到 MI-AC2100。
- `BUILD_NAME=MI-AC2100` 需要的预编译资源目录使用软链接复用 RM-AC2100。

## 关键假设

- MI-AC2100 和 RM-AC2100 的 NAND 分区布局、三 LAN 一 WAN、无线 EEPROM 偏移、MT7603/MT7615 组合一致。
- SWRT 当前 RM-AC2100 DTS 已能驱动同类硬件，MI-AC2100 首版不单独改交换机、HNAT、WiFi EEPROM 路径。
- OpenWrt 显示 MI 与 Redmi 的主要 DTS 差异是 LED GPIO 语义；SWRT 当前 RM-AC2100 DTS 未声明 gpio-leds，因此本次未单独补 MI LED 节点，避免引入未验证 LED 用户态行为。

## 后续验证

1. 运行静态检查：`git diff --check`。
2. 生成配置阶段确认 `MODEL_MIAC2100=y`、`CHIP_MT7603E/MT7615E` 选中。
3. 编译 `make swrt-mi-ac2100`，重点看预编译资源目录、DTS 生成、mt7603/mt7615 模块。
4. 上机后确认：串口启动、NAND 分区、LAN/WAN、2.4G/5G、复位键、LED 行为。
5. 若 LED 行为不对，再按 OpenWrt MI-AC2100 DTS 的 GPIO 6/8/10/12 映射补 SWRT LED 用户态逻辑。

## WireGuard 配置

默认构建预计包含 WireGuard：`release/src-rt/target.mak` 的 `GENERAL_BASE` 中已设置 `WIREGUARD=y`，`SWRT-MI-AC2100` 继承 `MT7621_NOUSB_BASE`，构建生成配置时会由 `release/src-rt/Makefile` 写入：

- `RTCONFIG_WIREGUARD=y`
- `CONFIG_WIREGUARD=m`

用户态工具由 `release/src/router/Makefile` 在 `RTCONFIG_WIREGUARD=y` 时编译 `wireguard-tools`，内核模块来自 `release/linux/mtk/linux-4.4.198/net/wireguard/`。

如果只想针对 MI-AC2100 自定义，优先改 `release/src-rt/target.mak` 的 `SWRT-MI-AC2100` profile，例如关闭：

```make
export SWRT-MI-AC2100 := $(MT7621_NOUSB_BASE) ... WIREGUARD=n ...
```

也可以构建时临时覆盖：

```sh
make swrt-mi-ac2100 WIREGUARD=n
```

不建议直接改生成后的 `release/src/router/.config` 或 kernel `.config`，这些文件在切换 target 或重新生成配置时会被覆盖。
